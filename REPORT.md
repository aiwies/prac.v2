# Что добавлено в проект Bookly

## 1. Что изменилось
Исходный вариант представлял собой одну страницу в `src/app/App.tsx`: регистрация, поиск по городу и отмена бронирования. Логика и разметка находились в одном месте, а данные исчезали после обновления браузера.

Теперь проект разделён на логические части и реализует учебный frontend-прототип из ТЗ.

## 2. Возможности сайта
- Регистрация и вход пользователя.
- Демонстрационные роли: USER, OWNER и ADMIN.
- Поиск по городу, категории и максимальной цене.
- Карточки объектов с типом жилья, рейтингом, ценой и удобствами.
- Модальное окно объекта с датами заезда/выезда и расчётом итоговой стоимости.
- Проверка запрета прошедших дат, неправильного диапазона дат и пересечения активных бронирований одного объекта.
- Запрет владельцу бронировать собственный объект.
- Личный кабинет: список бронирований, отмена без физического удаления, уведомления.
- Отзывы с оценкой от 1 до 5.
- Административная страница с пользователями, объектами и всеми бронированиями.
- Хранение данных в localStorage, поэтому после обновления страницы тестовые действия сохраняются.

## 3. Назначение папок
| Путь | Назначение |
|---|---|
| `src/app/App.tsx` | Корневой компонент: переключение экранов, глобальное состояние и обработчики сценариев. |
| `src/app/App.css` | Адаптивное оформление всех экранов и компонентов. |
| `src/entities/types.ts` | TypeScript-типы User, Listing, Booking, Review, Notification и AppData. |
| `src/features/ListingModal.tsx` | Сценарий просмотра объекта: бронирование, расчёт цены, проверка конфликта и отзыв. |
| `src/widgets/Header.tsx` | Навигация, роль пользователя, счётчик непрочитанных уведомлений, выход. |
| `src/widgets/ListingCard.tsx` | Повторно используемая карточка объекта. |
| `src/shared/ui/Button.tsx` | Общая кнопка с вариантами primary, ghost и danger. |
| `src/shared/lib/storage.ts` | Начальные данные, demo-аккаунты, localStorage и генерация идентификаторов. |
| `src/shared/lib/date.ts` | Форматирование дат/цен, расчёт ночей и проверка пересечения диапазонов. |

## 4. Демонстрация ролей
- ADMIN: `admin@bookly.ru` / `admin123`
- OWNER: `owner@bookly.ru` / `owner123`
- Обычного пользователя можно создать через вкладку «Регистрация».

## 5. Ограничение учебной версии
ТЗ описывает полноценный full-stack проект с Express, PostgreSQL, Prisma, JWT, Zod, Zustand и Tailwind. В присланном архиве был только Vite frontend без backend-проекта и без этих зависимостей. Поэтому в этой версии реализован работающий frontend-прототип с сохранением данных в браузере. Для production-варианта потребуется вынести работу с localStorage в REST API, подключить PostgreSQL/Prisma и перенести проверку прав на сервер.
